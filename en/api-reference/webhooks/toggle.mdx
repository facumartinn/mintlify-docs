# Toggle Webhook

Enable or disable a webhook without deleting it.

<Note>Requires `write:webhooks` scope.</Note>

## Request

```bash
POST /api/v1/webhooks/{id}/toggle
```

### Path Parameters

<ParamField path="id" type="string" required>
  The webhook ID
</ParamField>

### Body Parameters

<ParamField body="active" type="boolean" required>
  Set to `true` to enable, `false` to disable
</ParamField>

## Examples

### Disable Webhook

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/toggle" \
  -H "X-API-Key: zenflow_live_your_key" \
  -H "Content-Type: application/json" \
  -d '{"active": false}'
```

### Enable Webhook

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/toggle" \
  -H "X-API-Key: zenflow_live_your_key" \
  -H "Content-Type: application/json" \
  -d '{"active": true}'
```

### Response

```json
{
  "success": true,
  "data": {
    "id": "wh_abc123",
    "name": "Order Notifications",
    "active": false,
    "updated_at": "2024-01-14T15:30:00Z"
  }
}
```

## Use Cases

- **Maintenance**: Temporarily disable webhooks while updating your server
- **Debugging**: Pause webhook deliveries to investigate issues
- **Cost control**: Disable webhooks during high-traffic periods if needed
