# Rotate Webhook Secret

Generate a new secret for webhook signature verification.

<Note>Requires `write:webhooks` scope.</Note>

## Request

```bash
POST /api/v1/webhooks/{id}/rotate-secret
```

### Path Parameters

<ParamField path="id" type="string" required>
  The webhook ID
</ParamField>

## Example

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/rotate-secret" \
  -H "X-API-Key: zenflow_live_your_key"
```

### Response

```json
{
  "success": true,
  "data": {
    "id": "wh_abc123",
    "name": "Order Notifications",
    "secret": "whsec_new_secret_123..."
  }
}
```

<Warning>
  The new `secret` is only returned once. Store it securely and update your
  webhook verification code immediately.
</Warning>

## Best Practices

1. **Before rotating**: Prepare your server to accept both old and new secrets temporarily
2. **Rotate the secret**: Call this endpoint
3. **Update your server**: Deploy the new secret
4. **Verify**: Test that webhooks are being verified correctly
