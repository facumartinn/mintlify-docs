# Test Webhook

Send a test event to verify your webhook endpoint is working correctly.

<Note>Requires `write:webhooks` scope.</Note>

## Request

```bash
POST /api/v1/webhooks/{id}/test
```

### Path Parameters

<ParamField path="id" type="string" required>
  The webhook ID
</ParamField>

## Example

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/test" \
  -H "X-API-Key: zenflow_live_your_key"
```

### Success Response

```json
{
  "success": true,
  "data": {
    "status": "delivered",
    "response_status": 200,
    "response_time_ms": 150,
    "response_body": "OK",
    "event_sent": {
      "id": "evt_test_123",
      "event": "test",
      "created_at": "2024-01-14T15:30:00Z",
      "data": {
        "message": "This is a test webhook event",
        "webhook_id": "wh_abc123"
      }
    }
  }
}
```

### Failure Response

```json
{
  "success": false,
  "error": {
    "code": "delivery_failed",
    "message": "Webhook delivery failed",
    "details": {
      "response_status": 500,
      "response_body": "Internal Server Error",
      "response_time_ms": 1500
    }
  }
}
```

## Test Event Payload

The test event sent to your endpoint:

```json
{
  "id": "evt_test_123",
  "event": "test",
  "created_at": "2024-01-14T15:30:00Z",
  "data": {
    "message": "This is a test webhook event",
    "webhook_id": "wh_abc123"
  }
}
```

<Note>
  Test events are signed with your webhook secret, so you can verify your
  signature validation is working correctly.
</Note>
