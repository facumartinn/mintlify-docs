# Update Webhook

Update an existing webhook's configuration.

<Note>
  Requires `write:webhooks` scope.
</Note>

## Request

```bash
PUT /api/v1/webhooks/{id}
```

### Path Parameters

<ParamField path="id" type="string" required>
  The webhook ID
</ParamField>

### Body Parameters

All fields are optional. Only include fields you want to update.

<ParamField body="name" type="string">
  Webhook name
</ParamField>

<ParamField body="url" type="string">
  Webhook URL
</ParamField>

<ParamField body="events" type="array">
  Event types to subscribe to
</ParamField>

<ParamField body="description" type="string">
  Description
</ParamField>

<ParamField body="headers" type="object">
  Custom headers
</ParamField>

<ParamField body="active" type="boolean">
  Enable/disable the webhook
</ParamField>

<ParamField body="timeout_ms" type="integer">
  Request timeout
</ParamField>

<ParamField body="retry_count" type="integer">
  Number of retries
</ParamField>

## Example

```bash
curl -X PUT "https://api.zenflow.com/api/v1/webhooks/wh_abc123" \
  -H "X-API-Key: zenflow_live_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "events": ["order.created", "order.updated", "order.completed", "order.cancelled"],
    "timeout_ms": 60000
  }'
```

### Response

```json
{
  "success": true,
  "data": {
    "id": "wh_abc123",
    "name": "Order Notifications",
    "url": "https://example.com/webhooks/zenflow",
    "events": ["order.created", "order.updated", "order.completed", "order.cancelled"],
    "active": true,
    "timeout_ms": 60000,
    "updated_at": "2024-01-14T15:30:00Z"
  }
}
```
