## URL Base

Todas las peticiones a la API deben hacerse a:

```
https://api.zenflow.com/api/v1
```

## Autenticacion

La API de ZenFlow usa API Keys para autenticacion. Incluye tu API key en el header `X-API-Key`:

```bash
curl -X GET "https://api.zenflow.com/api/v1/orders" \
  -H "X-API-Key: zenflow_live_tu_api_key_aqui"
```

<Note>
  Las API keys tienen permisos (scopes) especificos. Asegurate de que tu key tenga los scopes necesarios para los endpoints que quieres usar.
</Note>

## Formato de Respuesta

Todas las respuestas se devuelven en formato JSON con una estructura consistente:

### Respuesta Exitosa

```json
{
  "success": true,
  "data": {
    // Datos de respuesta aqui
  }
}
```

### Respuesta de Error

```json
{
  "success": false,
  "error": {
    "code": "codigo_error",
    "message": "Mensaje de error legible"
  }
}
```

## Limites de Tasa (Rate Limits)

Las peticiones a la API tienen limites basados en la configuracion de tu API key:

| Tipo de Limite | Por Defecto |
|----------------|-------------|
| Por Minuto | 60 peticiones |
| Por Hora | 1,000 peticiones |
| Por Dia | 10,000 peticiones |

Los headers de rate limit se incluyen en todas las respuestas:

- `X-RateLimit-Limit`: Maximo de peticiones permitidas
- `X-RateLimit-Remaining`: Peticiones restantes
- `X-RateLimit-Reset`: Timestamp Unix cuando se reinicia el limite
