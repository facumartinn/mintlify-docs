# Probar Webhook

Envía un evento de prueba para verificar que tu endpoint de webhook está funcionando correctamente.

<Note>Requiere scope `write:webhooks`.</Note>

## Solicitud

```bash
POST /api/v1/webhooks/{id}/test
```

### Parámetros de Ruta

<ParamField path="id" type="string" required>
  El ID del webhook
</ParamField>

## Ejemplo

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/test" \
  -H "X-API-Key: zenflow_live_your_key"
```

### Respuesta Exitosa

```json
{
  "success": true,
  "data": {
    "status": "delivered",
    "response_status": 200,
    "response_time_ms": 150,
    "response_body": "OK",
    "event_sent": {
      "id": "evt_test_123",
      "event": "test",
      "created_at": "2024-01-14T15:30:00Z",
      "data": {
        "message": "Este es un evento de prueba de webhook",
        "webhook_id": "wh_abc123"
      }
    }
  }
}
```

### Respuesta de Fallo

```json
{
  "success": false,
  "error": {
    "code": "delivery_failed",
    "message": "Entrega de webhook fallida",
    "details": {
      "response_status": 500,
      "response_body": "Internal Server Error",
      "response_time_ms": 1500
    }
  }
}
```

## Payload del Evento de Prueba

El evento de prueba enviado a tu endpoint:

```json
{
  "id": "evt_test_123",
  "event": "test",
  "created_at": "2024-01-14T15:30:00Z",
  "data": {
    "message": "Este es un evento de prueba de webhook",
    "webhook_id": "wh_abc123"
  }
}
```

<Note>
  Los eventos de prueba están firmados con tu secret de webhook, así que puedes
  verificar que tu validación de firma está funcionando correctamente.
</Note>
