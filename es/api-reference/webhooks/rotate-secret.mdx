# Rotar Secret del Webhook

Genera un nuevo secret para la verificación de firma del webhook.

<Note>Requiere scope `write:webhooks`.</Note>

## Solicitud

```bash
POST /api/v1/webhooks/{id}/rotate-secret
```

### Parámetros de Ruta

<ParamField path="id" type="string" required>
  El ID del webhook
</ParamField>

## Ejemplo

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/rotate-secret" \
  -H "X-API-Key: zenflow_live_your_key"
```

### Respuesta

```json
{
  "success": true,
  "data": {
    "id": "wh_abc123",
    "name": "Notificaciones de Pedidos",
    "secret": "whsec_nuevo_secret_123..."
  }
}
```

<Warning>
  El nuevo `secret` solo se devuelve una vez. Guárdalo de forma segura y
  actualiza tu código de verificación de webhook inmediatamente.
</Warning>

## Mejores Prácticas

1. **Antes de rotar**: Prepara tu servidor para aceptar tanto el secret antiguo como el nuevo temporalmente
2. **Rotar el secret**: Llama a este endpoint
3. **Actualizar tu servidor**: Despliega el nuevo secret
4. **Verificar**: Prueba que los webhooks se están verificando correctamente
