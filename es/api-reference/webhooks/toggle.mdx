# Alternar Webhook

Habilita o deshabilita un webhook sin eliminarlo.

<Note>Requiere scope `write:webhooks`.</Note>

## Solicitud

```bash
POST /api/v1/webhooks/{id}/toggle
```

### Parámetros de Ruta

<ParamField path="id" type="string" required>
  El ID del webhook
</ParamField>

### Parámetros del Body

<ParamField body="active" type="boolean" required>
  Establece `true` para habilitar, `false` para deshabilitar
</ParamField>

## Ejemplos

### Deshabilitar Webhook

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/toggle" \
  -H "X-API-Key: zenflow_live_your_key" \
  -H "Content-Type: application/json" \
  -d '{"active": false}'
```

### Habilitar Webhook

```bash
curl -X POST "https://api.zenflow.com.ar/api/v1/webhooks/wh_abc123/toggle" \
  -H "X-API-Key: zenflow_live_your_key" \
  -H "Content-Type: application/json" \
  -d '{"active": true}'
```

### Respuesta

```json
{
  "success": true,
  "data": {
    "id": "wh_abc123",
    "name": "Notificaciones de Pedidos",
    "active": false,
    "updated_at": "2024-01-14T15:30:00Z"
  }
}
```

## Casos de Uso

- **Mantenimiento**: Deshabilita temporalmente los webhooks mientras actualizas tu servidor
- **Debugging**: Pausa las entregas de webhook para investigar problemas
- **Control de costos**: Deshabilita webhooks durante períodos de alto tráfico si es necesario
